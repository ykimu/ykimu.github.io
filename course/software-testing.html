Software Testing

ソフトウェアテスト：品質を「確かめる」技術

テストは「意図どおり動くことを示し、使われる前に欠陥を見つける」活動です。開発テスト（ユニット・コンポーネント・システム）からリリーステスト、ユーザテスト（アルファ・ベータ・受入れ）まで、代表的なテストを目的・実施の仕方・意識するポイント・注意点つきで整理しました。テストケース設計の技法（等価分割・境界値）、テスト駆動開発（TDD）に加えて、ブラックボックス／ホワイトボックスという設計の視点、UIテスト・フレーキーテストなど教科書の先にある現代的なテストまで収録。体験デモ、理解度クイズ付きです。

この教材は日本語のみで提供していますThis learning material is available in Japanese only.

体験デモ：等価分割と境界値を試してみよう

仕様：「入力は 10000〜99999 の整数（5桁）のみ有効」。値を入力すると、どの等価クラスに属するか、そして境界値かどうかを判定します。

値を入力するか、上のボタンで代表値を試してください。境界の前後（9999 / 10000、99999 / 100000）を試すと、境界値分析の考え方が体感できます。

おわりにIn closing

テストに向き合う心構えと、さらに学ぶためにHow to think about testing, and where to learn more

ソフトウェアテストの全体像（開発テスト・TDD・リリーステスト・ユーザテスト）を、これから学ぶ方向けにまとめた学習ガイドです。木村祐太のソフトウェア工学の授業教材の一部としてご覧いただけます。授業の詳細（評価方法・日程等）は最新のシラバスを確認してください。 A study guide to software testing (development testing, TDD, release testing, and user testing), for those new to it.
